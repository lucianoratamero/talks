HTML

O HTML (HyperText Markup Language) é uma linguagem de marcação. Se fizermos uma analogia, o HTML é o esqueleto de uma página. Ele contem o conteúdo da página e os dados necessários para o navegador exibir a página.

Boas práticas

  • declare a língua do conteúdo na tag html
  • deixar CSS e JS em outros arquivos
  • só use <div></div> e <span></span> caso precise declarar estilos
  • evite usar tags que geram efeitos visuais, como <em></em>, <b></b> e <br/>
  • só use um <h1></h1> por página, e não pule números (por exemplo, de h1 para h3)
  • se seu elemento não tiver texto, adicione uma descrição para leitores de tela
  • tudo que é clicável deve poder ser acessado pelo teclado com tab

CSS

O CSS (Cascading Style Sheets) é uma linguagem baseada em seletores e regras que definem como o HTML é exibido. Se o HTML é o esqueleto, o CSS é as características físicas - altura, cor, etc.

Boas práticas

  • pense no tamanho e peso de fonte
  • pense no contraste entre cores
  • use rem como medida para quase tudo, e px pro resto (bordas, por exemplo)
  • instale o WAVE Accessibility Extension e use ao menos uma vez a cada ciclo de desenvolvimento

Pausa

JS

O JS (JavaScript) é uma linguagem de programação baseada em protótipos. Ela usa de várias funcionalidades do navegador para tornar sua página mais interativa. Se o HTML é o esqueleto e o CSS é a forma física, o JS é os músculos, que dão vida ao que é estático.
